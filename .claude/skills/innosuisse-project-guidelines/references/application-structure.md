# Innosuisse Innovation Project Application Structure

## Submission Platform: Innolink

All applications are submitted through the Innolink online platform. The structure below reflects the main sections of the application form.

---

## Section 1: Organization Information

### Partners
| Field | Description |
|-------|-------------|
| Research Partner | Institution name, department, UID |
| Implementation Partner | Company name, UID, sector, size (FTE) |
| Additional Partners | If applicable |

### Team Composition
| Role | Required Information |
|------|---------------------|
| Project Lead (Research) | Name, institution, contact, FTE allocation |
| Project Lead (Implementation) | Name, company, contact, FTE allocation |
| Key Researchers | Names, expertise, FTE on project |
| Industry Experts | Names, roles, FTE on project |

### Mission Statement
- Brief description of each partner's mission (2-3 sentences)
- Relevant experience in the domain
- Previous collaboration history (if any)
- Why this partnership makes sense

---

## Section 2: Value Creation

### Business Model (~1 page)
- How will results be commercialized?
- Revenue model (licensing, product sales, services, SaaS)
- Pricing strategy
- Go-to-market approach
- Timeline to market entry

### Target Market
- Customer segments (who will buy?)
- Geographic focus (Switzerland first, then?)
- Market size estimates:
  - **TAM**: Total Addressable Market (CHF)
  - **SAM**: Serviceable Addressable Market (CHF)
  - **SOM**: Serviceable Obtainable Market (CHF)

### Competitive Advantage
- What makes this solution better than alternatives?
- Barriers to entry (IP, know-how, relationships)
- Time-to-market advantage
- Differentiation factors

### Economic Benefits
- Revenue potential for implementation partner (3-5 year projection)
- Job creation (direct and indirect in Switzerland)
- Export potential
- Contribution to Swiss competitiveness

---

## Section 4: Project Setup

### Work Packages (~2-3 pages)

Structure each WP as:
```
WP[X]: [Name] (Months X-Y)
  Lead: [Partner name]
  Effort: [Person-months]
  Objectives:
    1. [Measurable objective]
    2. [Measurable objective]
  Tasks:
    T[X].1: [Task description]
    T[X].2: [Task description]
  Deliverables:
    D[X].1: [Description] - Month [X] - [Type: Report/Prototype/Demo/Data]
    D[X].2: [Description] - Month [Y]
  Dependencies: [None / WP1, WP2]
```

### Milestones

| MS | Name | Month | Success Criteria | Go/No-Go |
|----|------|-------|------------------|----------|
| MS1 | [Name] | [X] | [Measurable criteria] | Yes/No |
| MS2 | [Name] | [X] | [Measurable criteria] | Yes/No |
| MS3 | [Name] | [X] | [Measurable criteria] | Yes/No |

**Guidelines:**
- 3-5 milestones for a 24-month project
- Spread milestones across project duration
- Include at least one go/no-go decision point
- Milestones should NOT cluster at end

### Gantt Chart
- Visual timeline showing all WPs
- Task dependencies
- Milestone markers
- Resource allocation per partner

## Section 6: Risk Analysis

### Risk Matrix

| ID | Risk | Category | Probability | Impact | Score | Mitigation |
|----|------|----------|-------------|--------|-------|------------|
| R1 | | Tech/Market/Org/Fin | H/M/L | H/M/L | | |
| R2 | | | | | | |

**Categories:**
- **Technical**: Technology risks, feasibility challenges
- **Market**: Demand uncertainty, competition, timing
- **Organizational**: Team, partnership, resources
- **Financial**: Costs, partner stability, funding

**Scoring:** H=3, M=2, L=1. Score = Probability x Impact.

## Section 9: Financial Annex (if applicable)

**Required for SMEs (<250 FTE):**
- Balance sheet (last 2 financial years)
- Profit/loss statement (last 2 years)
- Key financial ratios
- Confirmation of financial capacity to complete contribution

**Due diligence trigger:**
- If implementation partner with <250 FTE applies for >CHF 1M
